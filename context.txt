⭕ Exercice 00 : Polymorphism

Status : 🚀 EN COURS

Objectif :
Comprendre le polymorphisme avec fonctions virtuelles
Implémenter une hiérarchie Animal → Dog/Cat
Voir la différence entre virtual et non-virtual (WrongAnimal)

Fichiers à rendre :
Makefile, main.cpp, *.cpp, *.{h, hpp}

Classe Animal (Base) :
- Attribut protected : `std::string type`
- Méthode virtuelle : `virtual void makeSound() const`
- Méthode : `std::string getType() const` (retourne type)
- Constructeur/Destructeur avec messages

Classe Dog (hérite de Animal) :
- Initialise `type` à "Dog" dans le constructeur
- Override `makeSound()` → son de chien
- Constructeur/Destructeur avec messages différents

Classe Cat (hérite de Animal) :
- Initialise `type` à "Cat" dans le constructeur
- Override `makeSound()` → son de chat
- Constructeur/Destructeur avec messages différents

WrongAnimal et WrongCat (démonstration sans virtual) :
- Même structure qu'Animal/Cat
- MAIS `makeSound()` n'est PAS virtual
- Résultat : pointeur WrongAnimal* vers WrongCat appelle WrongAnimal::makeSound()

Code de test attendu :
```cpp
const Animal* meta = new Animal();
const Animal* j = new Dog();
const Animal* i = new Cat();
std::cout << j->getType() << std::endl;
std::cout << i->getType() << std::endl;
i->makeSound(); // Son de chat !
j->makeSound(); // Son de chien !
meta->makeSound(); // Son d'animal générique
```

Comportements attendus :
- `i->makeSound()` appelle Cat::makeSound() (grâce à virtual)
- `j->makeSound()` appelle Dog::makeSound() (grâce à virtual)
- Sans virtual (WrongAnimal), appelle toujours la méthode de la classe du pointeur

Concepts clés :
- Polymorphisme dynamique (runtime polymorphism)
- Mot-clé `virtual`
- Late binding vs early binding
- Destructeur virtuel obligatoire pour héritage

Status : À COMMENCER

Implémenter deep copy (copie profonde)

Fichiers à rendre :
Fichiers de l'exercice 00 + Brain.{h, hpp}, Brain.cpp

Classe Brain :
- Attribut privé : `std::string ideas[100]` (tableau de 100 idées)
- Constructeur/Destructeur avec messages
- Orthodox Canonical Form complète

Modification Dog et Cat :
- Ajout attribut privé : `Brain* brain`
- Constructeur : `brain = new Brain();`
- Destructeur : `delete brain;`
- Copy constructor : copie profonde (créer un nouveau Brain)
- Assignment operator : copie profonde (supprimer ancien, créer nouveau)

Test principal attendu :
```cpp
const Animal* animals[10];
for (int i = 0; i < 5; i++)
    animals[i] = new Dog();
for (int i = 5; i < 10; i++)
    animals[i] = new Cat();

for (int i = 0; i < 10; i++)
    delete animals[i]; // Doit appeler destructeurs Dog/Cat
```

Deep copy vs Shallow copy :
```cpp
Dog basic;
{
    Dog tmp = basic; // Copie profonde requise !
    // Si shallow copy : tmp et basic partagent le même Brain*
    // À la fin du scope, tmp détruit le Brain
    // basic a un pointeur vers Brain détruit → crash !
}
```

Comportements importants :
- Animal doit avoir destructeur **virtual** (`virtual ~Animal()`)
- Sans destructeur virtual : `delete (Animal*)dog` n'appelle pas ~Dog()
- Deep copy : créer un nouveau Brain, copier le contenu
- Pas de fuites mémoire (vérifier avec valgrind)

Concepts clés :
- Composition (has-a relationship)
- Allocation dynamique (new/delete)
- Deep copy vs shallow copy
- Destructeur virtuel obligatoire
- Memory leaks
- Rule of Three (constructor, destructor, assignment)

CONCEPTS THÉORIQUES À MAÎTRISER

Polymorphisme (Subtype Polymorphism) :
- Permet à un pointeur de classe base d'appeler les méthodes des classes dérivées
- Nécessite le mot-clé `virtual`
- Résolution à l'exécution (late binding / dynamic binding)

Virtual keyword :
```cpp
class Animal {
    virtual void makeSound() const; // Polymorphisme activé
};
class Dog : public Animal {
    void makeSound() const; // Override (pas besoin de "virtual" mais recommandé)
};

Animal* a = new Dog();
a->makeSound(); // Appelle Dog::makeSound() grâce à virtual
```

Sans virtual :
```cpp
class WrongAnimal {
    void makeSound() const; // PAS virtual
};
class WrongCat : public WrongAnimal {
    void makeSound() const;
};

WrongAnimal* w = new WrongCat();
w->makeSound(); // Appelle WrongAnimal::makeSound() (early binding)
```

Destructeur virtuel :
```cpp
class Animal {
    virtual ~Animal() {} // OBLIGATOIRE pour polymorphisme
};

Animal* a = new Dog();
delete a; // Sans virtual ~Animal(), ne détruit que la partie Animal
          // Avec virtual, appelle ~Dog() puis ~Animal()
```

Pure virtual function (= 0) :
```cpp
class Animal {
    virtual void makeSound() const = 0; // Classe abstraite
    // Pas d'implémentation dans Animal
};
// Animal a; → ERREUR : classe abstraite
```

Deep copy vs Shallow copy :
```cpp
// Shallow copy (MAUVAIS avec pointeurs)
Dog& Dog::operator=(const Dog& rhs) {
    brain = rhs.brain; // Les deux partagent le même Brain*
    return *this;
}

// Deep copy (CORRECT)
Dog& Dog::operator=(const Dog& rhs) {
    if (this != &rhs) {
        delete brain; // Supprimer l'ancien
        brain = new Brain(*rhs.brain); // Créer une copie
    }
    return *this;
}
```

Memory leaks :
- Oublier `delete` après `new` → fuite mémoire
- Vérifier avec `valgrind --leak-check=full ./program`
- Destructeur virtuel obligatoire pour delete via pointeur base

NOTE PÉDAGOGIQUE

Le CPP04 introduit le POLYMORPHISME DYNAMIQUE (Subtype Polymorphism) :
- Virtual functions pour résolution à l'exécution
- Pure virtual functions pour classes abstraites
- Gestion mémoire avec pointeurs polymorphes
- Deep copy avec allocation dynamique

Progression pédagogique :
- EX00 : Découverte du polymorphisme (virtual)
- EX01 : Polymorphisme + gestion mémoire + deep copy
- EX02 : Classes abstraites (pure virtual)

Différence avec CPP03 :
- CPP03 : Héritage simple, pas de virtual
- CPP04 : Polymorphisme avec virtual, gestion mémoire complexe
